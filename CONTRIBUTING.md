# Contributing to FlexProof

Thank you for your interest in contributing.

## Code of Conduct

Be respectful, constructive, and inclusive.

## How to Contribute

1. Fork the repository.
2. Create a feature branch from `main`.
3. Make your changes with clear commit messages.
4. Ensure all tests pass (`dotnet test`).
5. Open a pull request against `main`.

## Code Style

- .NET: follow standard C# conventions; nullable reference types enabled.
- TypeScript/Angular: follow the project `.editorconfig` and prettier config.

## Architecture Decision Records

Significant design changes should be proposed via an ADR in `docs/architecture/adr/`.
Use the template at `docs/architecture/adr/0000-template.md`.

## Security

If you discover a security vulnerability, please report it privately.
See [SECURITY.md](SECURITY.md).
